    What is an algorithm

  • Any well-defined computational procedure that takes in input in that form of some value or set of value and produces an output in the form of some value, or set of values is known as an algorithm.
  • Formerly defining, an algorithm is a sequence of computational steps that transforms the input into the output.

    Correctness and Characteristics of algorithm

  • An algorithm is said to be correct if, for every input instance, it halts with the correct output. We say that a correct algorithm solves the given computational problem.
  • An incorrect algorithm might not halt all on some input instances, or it might halt with an incorrect answer.

        • Order of growth of afunction

          Direct method

        • Ignore the lower order terms
        • Ignore the leading connstant.

        • Worst,average,best case and Asymptotic analysis

      • The time required by an algorithm falls under three types
        1. Best Case - Minimum time required for program execution : Ω Notation
        2. Average Case - Average time required for program exection : Θ Notation
        3. Worst case - Maximum time required for program execution : O Notation.

      • Big O Notation : Upper Bound

        It measures the worst case time complexity or the longest amount of time an algorithm can possibly take to complete.
        Mathematically,O(f(n))={g(n):there exists c > 0 and n > 0 such that f(n) <= c.g(n) for all n > nO.}
        "Little-o" (o()) notation is used to describe an upper-bound that cannot be tight.

      • Omega Ω Notation : Lower Bound

        It measures the best case time complexity or the best amount of time an algorithm can possibly take to complete.
        Mathematically,Ω(f(n))>={g(n):there exists c > 0 and nO > 0 such that c.g(n) <= f(n) for all n > nO.}
        "Little-omega" (Ω()) notation is used to describe an loose lower-bound.

      • Theta Θ Notation : Exact Bound

        The notation Θ(n) is the formal way to express both the lower bound and upper-bound of an algorithm's running time.
        Mathematically,Θ(f(n))={g(n) if and only if g(n) = O(f(n)) and g(n)= Θ(f(n)) for all n>nO.}
        "small theta" (Θ()) notation tells you both the loose upper bound of algorithm running time.
